# 题库与人物评分两层结构管理 — 实现总结

## 修改的文件清单（共 12 个）

### 数据库 Schema
1. **src/lib/schema.sql** — 新增 `question_sets` 和 `rating_groups` 表，`questions` 表加 `set_id` 列，`rating_targets` 表加 `group_id` 列

### 类型定义
2. **src/types/index.ts** — 新增 `QuestionSet` 和 `RatingGroup` 接口，`Question` 加 `set_id?: string`，`RatingTarget` 加 `group_id?: string`

### DB 适配器（3 个文件，三者同步修改）
3. **src/lib/db.ts** — 导出 8 个新函数（question_sets/rating_groups 各 4 个 CRUD）
4. **src/lib/db-d1.ts** — 实现 D1 版 question_sets/rating_groups 完整 CRUD；`getAllQuestions()` 增加可选 `setId` 参数；`createQuestion`/`updateQuestion` 支持 `set_id`；`getAllRatingTargets()` 增加可选 `groupId` 参数；`createRatingTarget`/`updateRatingTarget` 支持 `group_id`；所有涉及 `questions` 和 `rating_targets` 的 SQL 查询更新到包含新列
5. **src/lib/db-memory.ts** — 实现内存版 question_sets/rating_groups 完整 CRUD；`getAllQuestions()`/`getAllRatingTargets()` 增加过滤参数；`createQuestion`/`updateQuestion`/`createRatingTarget`/`updateRatingTarget` 支持新字段

### API 路由（4 个文件）
6. **src/app/api/admin/question-sets/route.ts** — 新建，支持 GET/POST/PUT/DELETE（各操作管理题目套）
7. **src/app/api/admin/rating-groups/route.ts** — 新建，支持 GET/POST/PUT/DELETE（各操作管理评分组）
8. **src/app/api/admin/questions/route.ts** — 更新，GET 支持 `?set_id=xxx` 过滤
9. **src/app/api/admin/ratings/route.ts** — 更新，GET 支持 `?group_id=xxx` 过滤

### 前端页面（2 个文件，完全重构）
10. **src/app/admin/questions/page.tsx** — 两层界面：
    - 顶层：卡片风格题目套列表（套名 + 创建时间）
    - 点击进入某套：显示套内题目（复用原有题目的 CRUD 逻辑）
    - 模态框创建/编辑题目套名称
11. **src/app/admin/ratings/page.tsx** — 两层界面：
    - 顶层：卡片风格评分组列表（组名 + 创建时间）
    - 点击进入某组：显示组内评分对象及其题目（复用原有逻辑）
    - 模态框创建/编辑评分组名称

## 关键设计决策
- 兼容性：现有 seed 数据的 `set_id`/`group_id` 均为 NULL，UI 和 API 无影响
- 级联删除：删除题目套时，该套下题目的 `set_id` 置 NULL（不删题）；删除评分组同理
- 风格一致：保持 Tailwind CSS、`btn`/`card`/`badge` 类名、`{ success, data, error }` 响应格式
- 无额外依赖：使用 `useEffect` + `fetch` 模式
- TypeScript 编译通过：`npx tsc --noEmit` 零错误
